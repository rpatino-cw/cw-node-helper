Full Workflow

How code reaches your teammates

From the moment you save a file to the moment a teammate runs the update script — every step, automated.

You — Local

Edit code in your editor

Make changes to get_node_context.py, fix bugs, add features. Save the file.

You — Terminal

Stage, commit, push

git add your changes → git commit with a message → git push to GitHub. Takes 10 seconds.

Automatic — GitHub Actions

CI pipeline triggers

GitHub detects the push and spins up 3 virtual machines — one for each Python version (3.9, 3.11, 3.13). Runs test_integrity.py on all three.

Automatic — Test Results

94 tests validate your code

Menu navigation, hotkeys, panels, transitions, rack math, config integrity — all checked in under 15 seconds. Red X if anything breaks.

You tag → Automatic

When you tag (git tag v6.3.0), GitHub builds a clean zip — no .env, no source/, no __pycache__. Published as a GitHub Release with auto changelog.

Teammates

They run update.sh

Teammates download the release zip and run bash update.sh. Their .env and bookmarks are preserved. Done in 5 seconds.

GitHub Actions detected push branch: main commit: f1d4c6e Spinning up test runners... ✓ test (3.9) 94 tests passed 17s ✓ test (3.11) 94 tests passed 10s ✓ test (3.13) 94 tests passed 11s All checks passed ✔ View run: github.com/.../actions/runs/...
Commit message convention: start with a type — feat: for features, fix: for bugs, docs: for documentation, test: for tests, ci: for pipeline changes.

Sharing & Distribution

How to give this app to a teammate

A step-by-step guide designed for anyone — no git knowledge required. Just follow the numbers.

1

Check if Python is installed

Open Terminal (search "Terminal" in Spotlight) and type:

python3 --version

If you see Python 3.10 or higher, you're good. If not, install it:

brew install python@3.13

No Homebrew? Visit brew.sh and paste the install command they show you.

2

Install the one dependency

The app only needs one Python package:

pip3 install requests

That's it. No frameworks, no databases, no Docker.

3

Download the release zip

Go to the GitHub Releases page and download cw-node-helper.zip:

https://github.com/your-org/node-helper/releases/latest

Or if you have the gh CLI:

gh release download --repo your-org/node-helper --pattern "*.zip"

If the repo is private, they'll need to run "gh auth login" first (one time).

4

Unzip and move to Documents

Double-click the zip to extract it, then move the folder:

mv ~/Downloads/cw-node-helper ~/Documents/cw-node-helper

Then go into the folder:

cd ~/Documents/cw-node-helper
5

Set up your API tokens

Copy the example config and fill in your credentials:

cp .env.example .env

Open the .env file in any text editor and paste your tokens. Follow ENV_GUIDE.txt for step-by-step instructions on getting each token.

Your .env file stays private. It is never uploaded or shared.

6

Run the app

Load your credentials and start:

source load_env.sh && python3 get_node_context.py

You'll see the main menu. Type a ticket number or pick an option. That's it!

7

Updating later

When a new version is released, download the new zip to your Downloads folder, then run:

bash update.sh

This replaces the app files but keeps your .env and bookmarks safe. You're up to date in 5 seconds.

Quick share message for Slack: "Hey! Download cw-node-helper.zip from the Releases page, unzip it, run pip3 install requests, copy .env.example to .env, add your Jira token, then run source load_env.sh && python3 get_node_context.py. That's it."

Future Migration

When your own repo is approved

Currently hosted as a branch on your-org/node-helper. When the dedicated your-org/cw-node-helper repo is approved, migrate in 3 commands.

1

Point to the new repo

git remote set-url origin git@github.com:your-org/cw-node-helper.git
2

Rename branch to main

git branch -M main
3

Push everything — code, history, and all version tags

git push -u origin main --tags
CI/CD carries over. The workflow files are already in the repo. Just update ci.yml to trigger on main instead of main and you're done.
