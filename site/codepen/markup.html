01 Getting Your API Keys

▸

Jira API Token (required)

This is how the tool authenticates with Jira on your behalf. It takes about 2 minutes.

  1. Open id.atlassian.com/manage-profile/security/api-tokens in your browser
  2. Click "Create API token"
  3. Name it something like cw-node-helper
  4. Click Create and copy the token immediately — you won't see it again
Jira tokens are tied to your account. Never share them. If compromised, revoke at the same URL and generate a new one.

NetBox API Token (optional but recommended)

NetBox enables rack elevations, cable tracing, device lookups, and neighbor navigation. Without it the tool still works — you just won't see hardware-level data.

  1. Open netbox.example.com/user/api-tokens/
  2. Click "Add a token"
  3. Copy the token
If you don't have NetBox access yet, ask your lead. The tool is fully usable with just Jira — add NetBox later anytime.

02 First Run Setup

▸

Prerequisites

  • Python 3.10+ — check with python3 --version
  • macOS or Linux terminal (Windows WSL works too)

Step-by-step

# 1. Install the one dependency pip3 install requests # 2. Copy the env template cp .env.example .env # 3. Edit with your credentials nano .env # or: code .env / open -e .env

Fill in your .env file:

JIRA_EMAIL=your.name@example.com JIRA_API_TOKEN=ATATT3xFfGF0abc123def... # Optional NETBOX_API_URL=https://netbox.example.com/api NETBOX_API_TOKEN=your-netbox-token-here

Launch

# Load env vars + start the app source load_env.sh && python3 get_node_context.py
You need to run source load_env.sh once per terminal session. It reads your .env and exports the variables. After that, just run python3 get_node_context.py to restart.

Troubleshooting

  • 401 Unauthorized — double-check JIRA_EMAIL and JIRA_API_TOKEN, no extra spaces
  • No module 'requests' — run pip3 install requests
  • command not found: python3 — install Python 3.10+ (brew install python3 on macOS)
  • No rack info — NetBox token missing or wrong; Jira features still work fine

03 CLI Commands & Usage

▸

Interactive Mode (default)

Just launch the app — you get a menu with numbered options, bookmark shortcuts, and a prompt. Type ticket keys, service tags, or hostnames directly at the menu.

# Launch interactive mode source load_env.sh && python3 get_node_context.py # Menu options: 1 Lookup ticket (DO-12345, HO-67890) 2 Node info (service tag or hostname) 3 Browse queue (DO or HO, by site + status) 4 My tickets (assigned to you) 5 Watch queue (live polling + notifications) 6 Rack map (visual location in data hall) 7 Bookmarks (manage saved shortcuts)

One-Shot Lookups

Skip the menu entirely — pass a ticket key, service tag, or hostname as an argument:

# By ticket key python3 get_node_context.py DO-12345 # By service tag python3 get_node_context.py 10NQ724 # By hostname python3 get_node_context.py d0001142

Queue Browsing

# Open tickets at your site python3 get_node_context.py queue --site US-SITE-01A --status open # HO tickets for a site python3 get_node_context.py queue --project HO --site US-EAST-03 # Just my tickets python3 get_node_context.py queue --mine

Node History

# All tickets for a service tag python3 get_node_context.py history 10NQ724 # All tickets for a hostname python3 get_node_context.py history d0001142

Background Watcher

# Watch for new tickets (macOS notifications + sound) python3 get_node_context.py watch --site US-SITE-01A --interval 45

04 Speed Tips — Make It Faster

▸

Built-in Caching

The tool caches aggressively so repeated lookups are instant:

  • Issue cache (100 entries) — revisiting a ticket doesn't re-fetch
  • JQL cache (200 entries, 60s TTL) — queue refreshes are fast
  • NetBox cache (50 entries) — device/rack data stays in memory
  • Parallel API calls — Jira + NetBox fetched simultaneously

Use Bookmarks

Save your most-used searches to slots a through e. Access them from the main menu with a single keypress.

# Inside a ticket or queue, press: [*] Bookmark this ticket or queue # From main menu, press: a All my tickets b Your site's open queue c My Verification tickets
Bookmarks persist across sessions in .cwhelper_state.json. Set them once, use them forever.

Skip the Menu

Type ticket keys or service tags directly at the main menu prompt — no need to press 1 first:

# At the "Select [1-7]" prompt, just type: DO-18739 — goes straight to the ticket 10NQ724 — looks up by service tag d0001142 — looks up by hostname

Turn Off Animations

The rack map and DH map have animated drawing. If you want instant rendering:

# Add to your .env or export manually: CWHELPER_ANIMATE=0

Install as a Global Command

Skip typing the full path every time:

# From the cw-node-helper directory: pip3 install -e . # Now works from anywhere: cwhelper DO-12345

05 Workflow Playbook — Tips & Tricks

▸

The Fastest Ticket Lifecycle

Start-to-close in as few keystrokes as possible:

# 1. Grab the ticket [a] Assign to yourself # 2. Start work [s] Moves ticket to "In Progress" # 3. View rack & connections while walking [r] Rack map — walking route from entrance [e] Rack elevation — see what's in the cabinet [n] Connections — which ports go where # 4. After the work is done [v] Move to Verification (add a comment if prompted) # 5. 24-48h later, close it [k] Close (with closing comment)

The Watcher Workflow

Best for shifts where you're waiting for tickets to land:

  1. Start the app and press 5 to start watching your site queue
  2. Work on other things — you'll get a macOS notification with sound when a new ticket arrives
  3. A card pops up with [g] Grab, [v] View, [s] Skip
  4. Grab it instantly — no one beats you to the ticket

Stale Verification Sweep

Press v at the main menu to see tickets stuck in Verification for 48+ hours. The SLA panel gives you a pre-written closing comment — just press [k] and paste.

Do a stale sweep at the start of every shift. It keeps the queue clean and boosts your team's SLA metrics.

Quick Checks Before Walking to a Rack

  1. [r] — see exactly where the rack is + walking route
  2. [n] — check which ports/cables you'll need to touch
  3. [e] — see what else is in the cabinet (PDUs, switches, other nodes)
  4. [o] — check the linked HO for vendor/RMA status
