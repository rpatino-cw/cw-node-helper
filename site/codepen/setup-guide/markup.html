Setup Guide
from Zero to Running

A step-by-step walkthrough to get the cw-node-helper tool running on your Mac. No experience needed. Just follow along.

Steps

  1. Before you start
  2. Open Terminal
  3. Install Homebrew
  4. Install Git
  5. Install Python
  6. Set up GitHub access
  7. Clone the repo
  8. Create your .env file
  9. Run the tool
  10. Staying up to date
  11. Troubleshooting
Step 01

Before You Start

Make sure you have these three things. If you're missing any, ask your manager or Ricky.

ⓘ What you need
  1. A Mac laptop — the tool runs in Terminal. Windows works too with WSL, but this guide covers Mac.
  2. Your CoreWeave email — something like yourname@coreweave.com
  3. A GitHub account — if you don't have one, sign up free at github.com/signup. Use your CoreWeave email so your team can identify you.
  4. Access to the CoreWeave GitHub org + the TopoWeave repo — this is the most important one and the most common blocker. See the check below.
⚠ Do this FIRST — check your repo access

Open this link in your browser while logged into GitHub:

github.com/coreweave/TopoWeave

What you should see: A page with files and folders (code). If you see this, you're good — keep going.

What means you DON'T have access:

  • "404 — This is not the page you're looking for" — you can't see the repo at all
  • "403 Forbidden" — you're logged in but not authorized
  • The page loads but says "Private repository" with no files

If you don't have access: You need to be added to the coreweave GitHub organization. Ask your manager or Ricky to invite your GitHub username. Once invited, you'll get an email from GitHub — you must click "Accept invitation" in that email before you can see the repo. This can take a few minutes.

✗ Don't skip this step
Everything else in this guide will fail if you don't have repo access. No amount of installing Git or SSH keys will help — the repo is private and requires org membership. Get this sorted first, then continue.
Step 02

Open Terminal

Terminal is a built-in app on every Mac. It lets you type commands instead of clicking around. Here's how to open it:

  1. Press Cmd + Space on your keyboard. This opens Spotlight Search — a little search bar in the middle of your screen.
  2. Type the word Terminal and press Enter.
  3. A black (or white) window appears with a blinking cursor. That's Terminal. You type commands here and press Enter to run them.
┌────────────────────────────────────────────────────────────────────────────────────┌ │ Terminal — bash — 80x24 │ ├────────────────────────────────────────────────────────────────────────────────────┤ │ │ │ yourname@MacBook-Pro ~ $ █ │ │ │ │ This is where you type commands. │ │ Type something, press Enter. That's it. │ │ │ └────────────────────────────────────────────────────────────────────────────────────┘
ⓘ Tip
Don't be scared of Terminal. You're just typing text and pressing Enter. If something goes wrong, you can always close the window and open a new one. Nothing breaks.
Step 03

Install Homebrew

Homebrew is like an app store for Terminal. It lets you install developer tools with one command. You need it to install Git and Python.

Check if you already have it

Terminal 
$ brew --version

If you see a version number like Homebrew 4.x.x, skip to the next step. If you see command not found, install it:

Install Homebrew

Copy and paste this entire line into Terminal and press Enter:

Terminal — paste this whole line 
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
⚠ It will ask for your password
Type your Mac login password. You won't see the characters as you type — that's normal. Just type it and press Enter. It may also ask you to press Enter to confirm. Just follow the prompts.

This takes a few minutes. When it finishes, it may print some "Next steps" telling you to run two more commands. If it does, copy and run those too. They look something like:

Terminal — only if Homebrew told you to 
$ echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
$ eval "$(/opt/homebrew/bin/brew shellenv)"

Verify it worked

Terminal 
$ brew --version
Homebrew 4.4.x   ← you should see something like this
Step 04

Install Git

Git is the tool that downloads code and keeps it in sync. Think of it like Google Drive for code.

Terminal 
$ brew install git

Verify

Terminal 
$ git --version
git version 2.x.x

Tell Git who you are

Replace the name and email with your actual name and CoreWeave email:

Terminal — use YOUR real name and email 
$ git config --global user.name "Your Name"
$ git config --global user.email "your.name@coreweave.com"
Step 05

Install Python

The tool is written in Python. You need Python 3.10 or newer.

Terminal 
$ brew install python3

Verify

Terminal 
$ python3 --version
Python 3.12.x   ← 3.10 or higher is fine

Install the one library the tool needs

Terminal 
$ pip3 install requests
Step 06

Set Up GitHub Access

You need to prove to GitHub that you're you so it lets you download the code. There are two ways. Option A is easier.

Install the GitHub CLI

Terminal 
$ brew install gh

Log in to GitHub

Terminal 
$ gh auth login

It will ask you a few questions. Answer like this:

Terminal — follow the prompts 
? What account do you want to log into?  GitHub.com
? What is your preferred protocol?        HTTPS
? Authenticate Git with your GitHub credentials? Yes
? How would you like to authenticate?     Login with a web browser

! First copy your one-time code: XXXX-XXXX
Press Enter to open github.com in your browser...
  1. It gives you a code (like ABCD-1234). Copy it.
  2. It opens your browser to GitHub. Paste the code when asked.
  3. Click Authorize.
  4. Go back to Terminal. It should say "Logged in".
✓ Done
Git is now connected to your GitHub account. You can download code.

Generate an SSH key

Replace the email with yours:

Terminal 
$ ssh-keygen -t ed25519 -C "your.name@coreweave.com"

It will ask where to save the file. Just press Enter to accept the default. It will ask for a passphrase. You can press Enter twice to skip it (no passphrase).

Copy your key

Terminal — this copies the key to your clipboard 
$ pbcopy < ~/.ssh/id_ed25519.pub

Nothing visually happens, but the key is now copied to your clipboard.

Add it to GitHub

  1. Open github.com/settings/ssh/new in your browser
  2. In the Title field, type something like Work Mac
  3. In the Key field, press Cmd + V to paste
  4. Click Add SSH key

Test it

Terminal 
$ ssh -T git@github.com
Hi yourname! You've successfully authenticated
✓ Done
Your SSH key is linked to GitHub. You can download code.
Step 07

Clone the Repo

"Cloning" means downloading the code to your computer. You only do this once.

Run these commands one at a time, pressing Enter after each:

Terminal — run each line one at a time 
# Go to your Documents folder
$ cd ~/Documents

# Download the code
$ git clone https://github.com/coreweave/TopoWeave.git
Cloning into 'TopoWeave'...
Receiving objects: 100% ... done.

# Go into the folder
$ cd TopoWeave

# Switch to the right branch (this is important!)
$ git checkout rpatino/cw-node-helper
Switched to branch 'rpatino/cw-node-helper'
ⓘ What just happened?
  • cd ~/Documents — moved into your Documents folder
  • git clone ... — downloaded all the code into a folder called TopoWeave
  • cd TopoWeave — went inside that folder
  • git checkout ... — switched to the branch that has the tool
✗ If you see "Permission denied", "404", or "Authentication failed"

This means one of two things:

  • You're not in the CoreWeave GitHub org — go back to Step 1 and get repo access first. No amount of retrying will fix this.
  • You have org access but GitHub doesn't know who you are on this computer — go back to Step 6 and set up GitHub authentication (gh CLI or SSH key).
Step 08

Create Your .env File

The .env file stores your personal API tokens so the tool can talk to Jira on your behalf. It stays on YOUR machine and is never shared.

8a. Copy the template

Terminal — make sure you're in the TopoWeave folder 
$ cp .env.example .env

8b. Get your Jira API token

  1. Open your browser and go to:
    id.atlassian.com/manage-profile/security/api-tokens
  2. Log in with your CoreWeave / Atlassian account if asked.
  3. Click the blue "Create API token" button.
  4. Give it a name. Type cw-node-helper and click Create.
  5. A popup shows your token. Click the copy icon next to it. Save this somewhere safe — you won't see it again!
⚠ Copy the token NOW
Once you close the popup, the token is gone forever. You'll have to create a new one. Paste it somewhere (like Notes) temporarily.

8c. Edit the .env file

The easiest way is to open it with TextEdit:

Terminal 
$ open -e .env

TextEdit opens. You'll see something like this:

.env — replace the placeholder values 
# Required
JIRA_EMAIL=your.name@coreweave.com        ← change this to YOUR email
JIRA_API_TOKEN=paste_your_jira_token_here  ← paste your token here

# Optional (leave as-is if you don't have NetBox)
NETBOX_API_URL=https://coreweave.cloud.netboxapp.com/api
NETBOX_API_TOKEN=paste_your_netbox_token_here

Replace the two red values with your real email and the token you copied. Example (not real values):

.env — example with fake values 
JIRA_EMAIL=john.doe@coreweave.com
JIRA_API_TOKEN=ATATT3xFfGF0abc123def456ghi789jklmno...

NETBOX_API_URL=https://coreweave.cloud.netboxapp.com/api
NETBOX_API_TOKEN=paste_your_netbox_token_here

Save the file (Cmd + S) and close TextEdit.

✗ Never share your .env file
It contains your personal API tokens. Don't send it in Slack, email, or commit it to Git. It's already in .gitignore so Git ignores it automatically.
Step 09

Run the Tool

You're almost there. The tool comes with a launcher script called start.sh that handles everything for you — credentials, updates, running the app. This is the only command you need to remember:

Terminal — make sure you're in the TopoWeave folder 
$ cd ~/Documents/TopoWeave
$ bash start.sh

You'll see the launcher menu:

start.sh — CW Node Helper Launcher 
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ┃  CW Node Helper                       ┃
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1  Run the app
  2  Update + Run        (get latest code, then launch)
  3  Update only         (download latest, don't run)
  4  First-time setup    (enter your credentials)
  5  Fix problems        (check everything is working)
  6  Open setup guide    (step-by-step instructions)

  d  Developer tools     (push, release, CI — advanced)
  q  Quit
ⓘ What each option does
  • 1 — Run the app — loads your credentials and launches the tool. If credentials aren't set up yet, it walks you through setup first.
  • 2 — Update + Run — downloads the latest code from GitHub, backs up your settings, then launches the app. Use this most of the time to stay up to date.
  • 3 — Update only — downloads updates but doesn't run the app.
  • 4 — First-time setup — guided wizard: asks for your email and Jira token, saves them. You do this once.
  • 5 — Fix problems — checks Python, dependencies, credentials, and the git repo. Auto-fixes what it can.
  • 6 — Open setup guide — opens this guide in your browser.
  • d — Developer tools — opens the git_menu.sh developer menu (push code, tag releases, check CI). You probably don't need this unless you're making code changes.

If this is your first time

Pick option 4 first to enter your credentials, then option 1 to run.

If the app is already set up

Just pick option 2 (Update + Run) — it handles everything in one step.

✓ If you see the launcher menu, you're good!
From here on out, bash start.sh is the only command you need. It runs the app, updates it, and fixes problems — all from one menu.

Running it again later

Every time you open a new Terminal window, just run these two commands:

Terminal — this is all you need to remember 
$ cd ~/Documents/TopoWeave
$ bash start.sh
Step 10

Staying Up to Date

When Ricky or someone else pushes updates, you need to download the latest code. The launcher handles this for you:

Terminal 
$ cd ~/Documents/TopoWeave
$ bash start.sh

Pick option 2 (Update + Run). It automatically:

  1. Backs up your credentials and bookmarks
  2. Downloads the latest code from GitHub
  3. Restores your credentials and bookmarks
  4. Launches the app
✓ That's it
Option 2 is the "do everything" button. Your credentials and bookmarks are always preserved — you never lose them during an update.

The developer menu (git_menu.sh)

There's also a separate git_menu.sh script for developers who need to push code, tag releases, or check CI. You can access it from the launcher by pressing d, or run it directly:

Terminal — developer tools only 
$ bash git_menu.sh

The developer menu includes:

  • Push — runs tests, stages changes, commits, and pushes to GitHub
  • Pull — downloads latest code
  • Status / Diff / Log — see what changed
  • Release — tag a version and trigger a build
  • CI check — see recent test results
⚠ Both scripts are branch-locked
Both start.sh and git_menu.sh are locked to the rpatino/cw-node-helper branch. You cannot accidentally switch branches, push to the wrong place, or break anything. If you try, the script blocks it.
Step 11

Troubleshooting

Something broke? Check the table below. Most problems have a one-line fix.

✗ "Permission denied" or "404 Not Found" when cloning
You don't have access to the GitHub repo. Ask your manager or Ricky to add your GitHub username to coreweave/TopoWeave. Then try the clone command again.
✗ "command not found: git"
Git isn't installed. Go back to Step 3 (Homebrew) and Step 4 (Git).
✗ "command not found: python3"
Python isn't installed. Go back to Step 5 (Python).
✗ "No module named 'requests'"
Run: pip3 install requests
✗ "Error: Jira returned 401 Unauthorized"
Your Jira email or API token is wrong. Open .env and double-check:
  • No extra spaces around the = signs
  • Email matches your Atlassian/Jira login
  • Token was copied correctly (no missing characters)
  • If the token is old, create a new one
✗ "fatal: not a git repository"
You're in the wrong folder. Run: cd ~/Documents/TopoWeave
✗ "error: pathspec 'rpatino/cw-node-helper' did not match"
The branch hasn't been fetched yet. Run: git fetch origin and then try git checkout rpatino/cw-node-helper again.
⚠ Still stuck?
Ask Ricky. Send him a screenshot of the error in Terminal — that helps a lot.
